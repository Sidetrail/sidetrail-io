This is a demo post showing the features and capabilites of the blog posts on this website, as well as a brief overview of how they work

Features

Formatting and Markdown

Each post is written in as a HTML file and supports the styling and formatting that is supported by modern HTML, for example

  • Lists

  • Code blocks

                    
const sum =  (x, y) => {
    return x+y;
}

  • Images

  • And more!

Embedded Code

Using JSFiddle, I can embed runable code into the blog post itself

How It Works

Each blog post is just an HTML file dynamically loaded from the public folder of the deployed application. The react code makes a request based on the URL for a HTML file and then sets the innerHTML to the payload

There is an index file that is fetched that contains the list of all blog posts and some summary information about them, below is an example payload containing a single blog post.

        
[
    {
        "id":"1",
        "title": "Demo Post",
        "headerImage":"/blog/header1.jpg",
        "tags":["test", "demo", "website"],
        "originalPostDate": "2021-07-13T17:53:36.315Z",
        "lastUpdatedDate": "2022-02-11T17:17:14.102Z",
        "summary": "This is a test post intended to prototype blogs.",
        "url":"test_post"
    }
]

This index file is the basis for blog landing page, and drives the functionality behind the tags, searching, and filtering as well. Each entry contains the high level information about the blog post and some meta-data
